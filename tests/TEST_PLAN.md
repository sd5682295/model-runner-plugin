# Model Runner Plugin - 完整测试计划

**版本**: v1.0  
**测试日期**: 2026-06-22

## 一、测试策略概览

### 测试金字塔

```
        /\
       /  \  E2E测试（10%）
      /    \
     /------\ 集成测试（30%）
    /        \
   /----------\ 单元测试（60%）
  /____________\
```

### 8 种测试方法

1. **单元测试** - 测试独立函数/类
2. **集成测试** - 测试模块间交互
3. **功能测试** - 测试用户操作流程
4. **回归测试** - 验证修复不破坏已有功能
5. **冒烟测试** - 快速验证核心功能
6. **边界测试** - 测试边界值和异常输入
7. **兼容性测试** - 跨平台/版本测试
8. **性能测试** - 响应时间和资源使用

## 二、测试环境

### 硬件/软件环境

- **操作系统**: Windows 10 Pro 19045
- **Obsidian 版本**: Latest
- **Node.js 版本**: v24.14.1
- **插件路径**: D:\Obsidian\vault\.obsidian\plugins\model-runner\
- **服务器路径**: {plugin}\server\

### 测试数据准备

- **测试 API Key**: 需要至少 2 个有效的 API Key
- **测试源**: 至少配置 3 个不同的源
- **测试模型**: gpt-3.5-turbo, claude-3-5-sonnet-20241022
- **测试端口**: 4000 (默认), 4001 (备用)

- 完整用户流程

### 阶段 5：冒烟测试（30 分钟）
- 快速验证核心功能
- 部署前最后检查
