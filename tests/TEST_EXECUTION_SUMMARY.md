## ⚠️ 当前问题

### 问题 1: TypeScript 模块导入

**现象**: Jest 无法直接导入 TypeScript 源文件

**原因**: 
- 项目使用 esbuild 打包成单个 main.js
- TypeScript 源文件不能直接被 Jest require
- 需要独立的模块导出

**解决方案**:
1. **方案 A** (推荐): 为每个模块添加独立的编译输出
   - 创建 `build/` 目录
   - 编译每个 .ts 文件为独立的 .js
   - 测试导入 build/ 下的文件

2. **方案 B**: 使用 ts-jest 直接测试 TypeScript
   - 配置 ts-jest 正确处理 Obsidian API
   - 需要更复杂的配置

3. **方案 C**: 手动测试核心功能
   - 直接在 Obsidian 中测试
   - 更接近真实环境
