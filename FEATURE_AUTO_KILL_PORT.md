# ✨ 新功能：一键停止占用端口的进程

**功能版本**：Phase 2 v0.2.2

## 🎯 功能概述

当启动服务器时，如果端口 4000 被旧进程占用，插件会**自动检测并停止**旧进程，无需手动操作！

---

## ✨ 核心功能

### 1. 自动端口检查
- ✅ 启动前检查端口 4000 是否被占用
- ✅ 跨平台支持（Windows/Mac/Linux）
- ✅ 实时反馈占用状态

### 2. 一键停止旧进程
- ✅ **自动停止**占用端口的旧进程
- ✅ 无需手动运行命令
- ✅ 友好的通知提示

### 3. 智能重试
- ✅ 停止旧进程后自动等待 500ms
- ✅ 确保端口完全释放后再启动
- ✅ 失败时给出明确的错误提示

## 🔧 技术实现

### 跨平台端口检查

**Windows**：
```typescript
exec('netstat -ano | findstr :4000', (err, stdout) => {
  // 检查是否有输出
});
```

**Mac/Linux**：
```typescript
exec('lsof -i :4000', (err, stdout) => {
  // 检查是否有输出
});
```

### 跨平台进程停止

**Windows**：
```typescript
// 1. 查找占用端口的进程
exec('netstat -ano | findstr :4000', (err, stdout) => {
  const pids = extractPIDs(stdout);
  
  // 2. 强制停止所有进程
  pids.forEach(pid => {
    exec(`taskkill /PID ${pid} /F`);
  });
});
```

**Mac/Linux**：
```typescript
// 1. 查找占用端口的进程
exec('lsof -ti:4000', (err, stdout) => {
  const pids = stdout.split('\n');
  
  // 2. 强制停止所有进程
  pids.forEach(pid => {
    exec(`kill -9 ${pid}`);
  });
});
```

### 错误处理

```typescript
try {
  await this.killPort(4000);
  new Notice('✅ 已停止旧进程，正在启动...');
} catch (error) {
  new Notice('❌ 无法停止旧进程，请手动停止');
  return false; // 阻止启动
}
```
