## 🎯 功能演示

### 1. 自动启动
1. 进入设置页面
2. 打开 **自动启动** 开关
3. 重启 Obsidian
4. 服务器会自动启动

### 2. 切换 API 源
1. 进入设置页面
2. 在 **当前源** 下拉框中选择源
3. 如果服务器正在运行，会提示重启
4. 重启服务器后新源生效

### 3. 配置端口
1. 进入设置页面
2. 修改 **服务器端口**
3. 输入新端口（如 5000）
4. 重启服务器使新端口生效

### 4. 启用/禁用源
1. 进入设置页面
2. 在 **可用的源** 列表中
3. 切换任意源的开关
4. 配置立即保存到 config.json

---

## ⚠️ 重要说明

### 配置加载时机
- 配置在 **首次启动服务器时** 加载
- 如果启动前未启动过服务器，配置为空
- 启动后配置会自动加载并显示

### 配置修改生效
- **立即生效**：源启用/禁用
- **需要重启**：端口、当前源
- **下次启动生效**：自动启动

### 错误处理
- 配置文件不存在时会提示
- 端口范围验证（1024-65535）
- 重启次数验证（0-10）
- 切换不存在的源会报错

## 📚 配置文件格式

### config.json 结构
```json
{
  "apiKey": "your-api-key",
  "sources": {
    "mytokenland": {
      "provider": "mytokenland",
      "baseURL": "https://api.mytokenland.com",
      "models": ["claude-3-5-sonnet-20241022", ...],
      "enabled": true
    },
    "openai": {
      "provider": "openai",
      "baseURL": "https://api.openai.com/v1",
      "models": ["gpt-4", ...],
      "enabled": false
    }
  },
  "currentSource": "mytokenland",
  "port": 4000,
  "maxRetries": 3,
  ...
}
```

### 插件设置文件
存储在 `.obsidian/plugins/model-runner/data.json`：
```json
{
  "autoStart": false,
  "port": 4000,
  "showNotifications": true,
  "autoRestart": true,
  "maxRestartAttempts": 3
}
```
