## 🎯 测试策略

### 测试金字塔

```
        /\
       /  \       E2E Tests (手动)
      /____\      
     /      \     Integration Tests (自动化)
    /        \    
   /__________\   Unit Tests (自动化)
```

**测试覆盖率目标**：
- 单元测试：≥ 80% 代码覆盖率
- 集成测试：100% 关键路径覆盖
- 功能测试：100% 功能点覆盖
- 回归测试：100% 核心功能覆盖

## 📊 8种测试方法

### 1. 单元测试 (Unit Testing)
**目标**：验证每个独立模块的正确性

**测试对象**：
- ProcessManager.ts
- ModelRunnerView.ts
- ConfigManager.ts (Phase 2)
- HealthChecker.ts (Phase 3)

**测试内容**：
- 每个方法的输入输出
- 边界条件
- 异常处理

### 2. 集成测试 (Integration Testing)
**目标**：验证模块间协作

**测试场景**：
- main.ts + ProcessManager 交互
- ProcessManager + server.js 进程通信
- ModelRunnerView + ProcessManager 数据流
- 配置读写 + 进程启动流程

### 3. 功能测试 (Functional Testing)
**目标**：验证所有用户功能

**测试范围**：
- 启动/停止/重启服务器
- 日志显示和清空
- 状态栏更新
- 命令面板操作
- 侧边栏交互

### 4. 冒烟测试 (Smoke Testing)
**目标**：快速验证基本功能

**测试内容**：
- 插件能否加载
- 服务器能否启动
- 日志能否显示
- 能否停止服务器

### 5. 回归测试 (Regression Testing)
**目标**：确保修改不破坏现有功能

**测试内容**：
- 每次代码修改后重跑所有测试
- 验证核心功能不受影响

### 6. 性能测试 (Performance Testing)
**目标**：验证性能指标

**测试内容**：
- 启动时间 < 3秒
- 日志渲染性能
- 内存占用
- CPU 使用率

### 7. 安全测试 (Security Testing)
**目标**：验证安全性

**测试内容**：
- API Key 不泄露到日志
- 路径遍历攻击防御
- 命令注入防御

### 8. 用户验收测试 (UAT)
**目标**：用户实际使用场景

**测试内容**：
- 完整用户工作流
- 可用性和易用性
- 错误提示友好性
