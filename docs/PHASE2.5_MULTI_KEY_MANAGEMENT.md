### 关键功能

#### 1. Key 脱敏显示

```typescript
private maskKey(key: string): string {
  if (key.length <= 8) return '••••••••';
  return key.substring(0, 8) + '•'.repeat(Math.min(key.length - 8, 20));
}
```

**示例**：
- `N1HKX3SR-QM5J-DHFC-AJQU-JS78VNMHCYVT` → `N1HKX3SR••••••••••••••••••`
- `sk-9b36ccf10ce5410a` → `sk-9b36c•••••••••••`

#### 2. Key 测试连接

```typescript
private async testKey(key: string, statusEl: HTMLElement): Promise<void> {
```

#### 3. Key 数组管理

```typescript
// 添加
this.apiKeys.push(newKey);

this.apiKeys[index] = newKey;

// 删除
this.apiKeys.splice(index, 1);

await configManager.save(config);
```

#### 4. 保护机制

```typescript
// 至少保留一个 Key
if (this.apiKeys.length === 1) {
  new Notice('❌ 至少需要保留一个 Key');
  return;

// 防止重复添加
if (this.apiKeys.includes(newKey)) {
```

## 🎯 使用场景

### 场景 1：备份 Key

**问题**：主 Key 失效时服务中断

**解决方案**：
1. 点击"管理 Keys"
2. 点击"➕ 添加新 Key"
3. 输入备用 Key
4. 点击"测试"验证
5. 保存

**效果**：主 Key 失效时自动切换到备用 Key

### 场景 2：提高并发

**问题**：单个 Key 有速率限制

**解决方案**：
1. 添加多个 Key（如 3 个）
2. 选择"顺序轮询"策略
3. 保存

**效果**：请求自动分散到 3 个 Key，并发能力提升 3 倍

### 场景 3：分散配额

**问题**：单个 Key 配额有限

**解决方案**：
1. 购买多个账号的 Key
2. 全部添加到同一个源
3. 轮询使用

**效果**：配额累加，使用时间延长

### 场景 4：测试 Key 有效性

**问题**：不确定 Key 是否有效

**解决方案**：
1. 打开"管理 Keys"
2. 点击每个 Key 的"测试"按钮
3. 查看测试结果

**效果**：
- ✅ 正常 (12 个模型) - Key 有效
- ❌ 失败 (401) - Key 无效
- ❌ 超时 - 网络问题

## 📊 数据格式

### config.json 结构

```json
{
  "sources": [
    {
      "id": "yunyi",
      "name": "云驿",
      "baseUrl": "https://yunyi.yun/claude/v1",
      "apiKeys": [
        "N1HKX3SR-QM5J-DHFC-AJQU-JS78VNMHCYVT",
        "sk-backup-key-2",
        "sk-backup-key-3"
      ]
    }
  ]
}
```

**说明**：
- `apiKeys` 是**数组**
- 支持任意数量的 Keys
- 按顺序轮询使用
- 与 server.js 完全兼容
